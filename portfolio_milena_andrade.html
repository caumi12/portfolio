Arquitetura de Atribuição de Receita no SFMC

A operação de e-mail gerava vendas que ninguém conseguia rastrear. Para resolver isso, projetei uma arquitetura de atribuição com janela de 5 dias pós-clique. O resultado foi uma base consolidada e pronta para análise, reduzindo o trabalho manual do time de Dados e entregando indicadores confiáveis para a diretoria.

Arquitetura construída no SFMC

DATA EXTENSION Cliques — limpeza de dados

SQL no Query Studio filtra cliques nas campanhas. Ponto de entrada limpo para toda a cadeia de atribuição.

Chave composta JobID + Data_Venda

Mesma campanha pode converter em dias diferentes. A chave composta garante que cada linha represente uma conversão única, preservando a granularidade do dado.

Regra de negócio — janela de 5 dias

Query valida se a venda ocorreu dentro de 5 dias após o clique. Lógica documentada, auditável e replicável pelo time de dados sem depender de memória ou interpretação manual.

DATA EXTENSION KPIs

Tabela final com receita, pedidos, ROI e CAC por campanha, gerada automaticamente pelo Automation Studio. Time de dados recebe o arquivo pronto, sem cruzamentos manuais.

Pipeline de dados no Automation Studio

Sequência de execução — Query Studio → Data Extensions
1
DE_Cliques_Limpos — SQL filtra cliques reais das últimas 24h. Garante que só intenção explícita entre na janela.
2
DE_Vendas_CRM — cruza cliques com pedidos pelo subscriber key. Aplica a regra de negócio: venda deve ocorrer em até 5 dias após o clique.
3
DE_KPIs_Campanha — consolida por chave composta JobID + Data_Venda. Cada linha = uma conversão única. Receita, pedidos, ROI e CAC por campanha e por dia.
4
Automation Studio executa a sequência diariamente. BI recebe dado limpo e auditável.

WhatsApp CRM: Novo canal com ROAS 40x

O canal existia, mas não era utilizado no CRM. Apenas o time de vendas disparava mensagens, mas sem segmentação, sem copy personalizada e sem horário testado.

O que foi feito

Diagnóstico de persona

Mapeamento do perfil: momento de jornada, gatilhos de decisão e sensibilidade a tom e urgência. Base para todas as decisões de copy e segmento.

Hipóteses testadas

Cada disparo documentado como teste com variável isolada: segmento vs. copy vs. horário. Aprendizado acumulado em 15 disparos: público é a alavanca nº 1.

Garantia de Opt-Out

Incluí uma saída amigável e clara em todas as mensagens para blindar o canal contra bloqueios e preservar a reputação do número corporativo.

Régua segmentada por perfil

Cada público com abordagem e copy própria. Webhook Klaviyo → OctaDesk fecha o ciclo: resposta cai direto no SAC.

Funil de Aquisição Omnichannel

O Problema: conectar leads de 150 lojas físicas aos seus respectivos grupos de WhatsApp de forma automática e rastreável. O desafio era direcionar cada lead captado para o grupo de WhatsApp correto, por região, de forma automática, com rastreamento completo e sem depender de plataformas externas.

O que foi feito

Captura e segmentação

Landing Page hospedada no Salesforce com captura em tempo real e seleção de loja pelo próprio lead. Dados gravados via UpsertData na Data Extension.

Redirecionamento automático

AMPscript consulta o Mapa de Grupos por LojaCodigo e redireciona para o link de WhatsApp correto após 3 segundos do cadastro.

Lista de espera com cronômetro

Página de espera até data de lançamento para gerar antecipação. Após a data do lançamento: redirecionamento automático para o grupo da unidade.

Rastreamento completo

GTM com evento form_conversion no DataLayer. UTMs por canal (Instagram, PDV, Ads).

Newsletter de Relacionamento
